`Help`

`Use one SELECT or CTE query. Use @param placeholders for filters, then define optional parameter metadata as JSON.`

`Supported Parameter Types`

  • text - `free text input`
  • int - `whole number input`
  • number - `decimal number input`
  • date - `date input; defaults can use today or -30d`
  • datetime - `date and time input`
  • lookup - `select input from JSON options`
  • lookup_table - `select input from a table with id and iname columns`
  • lookup_model - `select input from a model's listSelectOptions()`
  • lookup_sql - `select input from SQL returning id and iname`
  • lookup_tpl - `select input from a .sel template`

`Copyable SQL Example`

SELECT id, fname, lname, email, access_level, add_time
FROM users
WHERE (@access_level IS NULL OR access_level = @access_level)
  AND (@s IS NULL OR email LIKE '%' + @s + '%' OR fname LIKE '%' + @s + '%' OR lname LIKE '%' + @s + '%')
ORDER BY add_time DESC
