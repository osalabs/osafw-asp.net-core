`Help`

`Use one SELECT or CTE query. Use @param placeholders for filters, then define optional parameter metadata as JSON. Supported parameter types are text, int, number, date, datetime, and lookup.`

`Example SQL`

SELECT id, fname, lname, email, access_level, add_time
FROM users
WHERE (@access_level IS NULL OR access_level = @access_level)
  AND (@s IS NULL OR email LIKE '%' + @s + '%' OR fname LIKE '%' + @s + '%' OR lname LIKE '%' + @s + '%')
ORDER BY add_time DESC
