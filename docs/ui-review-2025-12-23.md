# UI Review Notes (2025-12-23)

## Current baseline
- Pages load Bootstrap 5 plus the default `site.css`, and conditionally append a theme override, so core styling must look modern on its own before optional themes layer on top.【F:osafw-app/App_Data/template/layout/sys_head.html†L14-L18】
- The default theme keeps the body flat white with lightly bordered cards and minimal shadowing, which reads functional but not polished for enterprise dashboards.【F:osafw-app/wwwroot/assets/css/site.css†L4-L166】
- Custom button styling and table density lean toward compact layouts, but the visual hierarchy between primary and neutral actions is weak in the default palette.【F:osafw-app/wwwroot/assets/css/site.css†L73-L95】【F:osafw-app/wwwroot/assets/css/site.css†L304-L356】

## Issues to fix
1. **Clean up CSS correctness** – the `a[disabled]` selector is malformed, so disabled links still look interactive; fix the rule so pointer suppression and disabled coloring apply consistently.【F:osafw-app/wwwroot/assets/css/site.css†L96-L100】
2. **Strengthen card and page framing** – introduce subtle shadows and consistent padding for `.fw-card` blocks (list filters, forms, view panes) to match the more modern treatment seen in optional themes, while keeping borders lean.【F:osafw-app/wwwroot/assets/css/site.css†L153-L166】【F:osafw-app/wwwroot/assets/css/theme20.css†L63-L77】【F:osafw-app/wwwroot/assets/css/theme30.css†L84-L96】
3. **Improve table readability** – default table rows use small fonts and nearly edge-to-edge cells, so increase base padding/line-height for non-dense views and tone down header background; ensure sticky headers and selected-row highlights remain visible in dark mode.【F:osafw-app/wwwroot/assets/css/site.css†L304-L356】
4. **Refine bulk-action and pagination affordances** – the multi-row action bar and pagination select butt against surrounding elements; add spacing, consistent button radii, and align the select with pagination pills to reduce clutter.【F:osafw-app/wwwroot/assets/css/site.css†L510-L536】
5. **Clarify form spacing and alignment** – `.form-row` only adds bottom padding and labels right-align at ≥768px, which can feel cramped in two/three-column grids; add vertical rhythm, keep right-aligned labels, and standardize helper text/validation spacing.【F:osafw-app/wwwroot/assets/css/site.css†L544-L599】
6. **Balance multi-value fields and attachments** – the fixed height for multi-value lists and tight attachment tiles work on desktop but overflow on narrow screens; add responsive max-heights and gutters to keep content scannable.【F:osafw-app/wwwroot/assets/css/site.css†L593-L655】

## Rework opportunities
- **Elevate default typography** – adopt the cleaner font stacks and size tuning used in `theme20`/`theme30` so the base theme feels contemporary without requiring a theme toggle.【F:osafw-app/wwwroot/assets/css/theme20.css†L52-L77】【F:osafw-app/wwwroot/assets/css/theme30.css†L68-L96】 
- **Unify action button hierarchy** – reserve solid fills for primary/destructive actions and keep neutral buttons outlined or subtle-filled, ensuring hover/focus states remain visible in both light/dark modes.【F:osafw-app/wwwroot/assets/css/site.css†L58-L78】
- **Responsive polish** – sidebar and card layouts are sticky by default; add small-screen transitions (e.g., smoother collapse, shadow overlay) and ensure form sections stack with consistent gaps when columns wrap.【F:osafw-app/wwwroot/assets/css/site.css†L152-L220】【F:osafw-app/wwwroot/assets/css/site.css†L499-L620】

## Quick wins
- Fix the disabled-link rule and refresh default card/table padding for immediate readability gains.
- Borrow the shadow + radius tokens from `theme20`/`theme30` to modernize the default theme without breaking existing overrides.
- Adjust form label alignment and spacing to reduce visual noise on dense edit screens.
