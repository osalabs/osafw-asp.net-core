## Decisions - why
- Preserved `is_fwupdates_auto_apply` as a compatibility redirect gate; it now controls only whether Home redirects to the pending notice.
- Kept `/Dev/Configure/(PendingUpdates)` read-only and public so anonymous dev/bootstrap visitors can see the pending-update notice without executing SQL.
- Used `/Dev/Manage/(ApplyPendingFwUpdates)` for execution because `/Admin/FwUpdates` already renders POST buttons with XSS tokens.
- Hardened all FwUpdates DevManage mutations with `enforcePost()` since the UI already posts tokens for these actions.

## Pitfalls - fixes
- The old custom action route was method-agnostic; deleting `ApplyUpdatesAction` and moving execution behind `enforcePost()` closes the GET path.
- Login already validated `gourl` on save, but the login form did not preserve safe GET targets; added a validated hidden field.
